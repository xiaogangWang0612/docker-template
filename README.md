# 本系统是一个模板工程

## 相关说明
````
1、本案例代码主要是为方便实现Docker镜像管理写的, 如果没有使用Docker就不必看了
2、本案例是为了给正在使用Docker服务发布系统, 但是又不想自己搞本地Registry、想用云镜像的同仁做参考的 
3、使用云镜像服务也有好处(不管是腾讯还是阿里), 避免了自己做Registry后存在的网络风险,如是内网可以忽略
4、可以最大化节省Docker镜像编译的工作, 合理利用云的触发机制
````
## 其他说明
````
1、本案例中pom文件有多个plugin主要是为了实现代码源码打包文件、配置文件、依赖lib文件分离, 如果没有需求, 可以随意更改
2、本案例中核心参考代码主要还是看Dockerfile中的代码
3、本案例经过测试, 不管是阿里云还是腾讯云的镜像库都没有问题
4、本案例采用的最终制作镜像比较随意是因为比较小, 实际使用版本请按需选择
````
## 其他建议
````
使用docker发布系统时候, 强烈建议将docker服务产生的文件挂在到宿主机, 避免日志丢失
````
